1. 项目架构概览

1.1 系统角色与进程

整个系统由 三个独立进程 协作完成：

+---------------------------+       TCP 8888        +---------------------+
|   server_app              | ◄───────────────────► |   client_app         |
|   (云端曲库服务端)         |    sendfile 零拷贝     |   (车载播放引擎)     |
|   — 扫描 media/ 目录 |                        |   — 预加载下载  |
|   — sendfile 推送 MP3|                        |   — fork/mplayer 播放|
+---------------------------+                        +---------+-----------+
                                                                  | 命名管道
                                                                  +─ /tmp/ui_cmd
                                                                  +─ /tmp/mplayer_cmd
                                                                  +─ /tmp/carradio_status
                                                          +-------+-----------+
                                                          |   ui_qt           |
                                                          |   (Qt 5 图形界面) |
                                                          |   — 歌单浏览 |
                                                          |   — 播放控制 |
                                                          +-------------------+

1.2 数据流

方向内容协议
UI → clientplay:5, next, pause, volume 10命名管道 /tmp/ui_cmd
client → server歌曲编号 "5"TCP 8888
server → clientMP3 二进制流sendfile (TCP)
client → UIPLAYING:5, DOWNLOADING, IDLE命名管道 /tmp/carradio_status
client → mplayerpause, seek 100 2命名管道 /tmp/mplayer_cmd

4. server/server.c — 云端曲库服务端

4.1 整体职责

接收客户端 → 解析歌曲编号 → open() 打开 MP3 → sendfile() 零拷贝发送

4.2 全局变量

  • char **song_table — 歌曲路径指针数组 ["media/ch1/a.mp3", ...]
  • int song_count — 歌曲总数

4.3 函数详解

static void on_signal(int sig)

信号处理：SIGPIPE 忽略（避免客户端断开导致退出），SIGINT/SIGTERM 优雅关闭。

static int count_mp3(const char *subdir)

API作用
opendir()打开目录流
readdir()读取下一项目录条目
strcasecmp()不区分大小写比较字符串
closedir()关闭目录流

static void fill_mp3(const char *subdir, char **table, int offset)

将目录下所有 .mp3 完整路径填入数组。示例：fill_mp3("media/ch1", song_table, 0) → song_table[0] = "media/ch1/song.mp3"

static int cmp(const void *a, const void *b)

qsort 排序回调，按文件名升序排列。

static void build_song_table()

扫描 media/ch1, media/ch2, media/ch3 构建完整路由表。

算法：第一遍 count_mp3 统计总数 → calloc 分配 → 第二遍 fill_mp3 填充 → qsort 排序。

int main() — 主程序

三个阶段：

  1. 初始化 — setlinebuf, signal 注册, build_song_table
  2. TCP 服务端创建 — socket, bind(8888), listen(10)
  3. 主循环 while(1) — accept, recv 编号, open MP3, sendfile 发送, close

while(1) 保证 server 永不退出，持续接受新连接。

4.4 sendfile 零拷贝

ssize_t sendfile(int out_fd, int in_fd, off_t *offset, size_t count);

传统路径：硬盘 → 内核缓冲区 → 用户缓冲区 → 内核Socket缓冲区 → 网卡

sendfile路径：硬盘 → 内核缓冲区 → 网卡（减少两次拷贝和上下文切换）

5. client/client.c — 车载播放引擎客户端

5.1 整体职责

连接服务器 → 下载 MP3 → fork 子进程播放 → select 监听 UI 指令

这是整个系统 最核心的模块，实现了"预加载下载"和"可中断控制"。

5.2 宏定义

宏路径描述
UI_CMD_PIPE/tmp/ui_cmdUI → client 指令通道
MPLAYER_CMD_PIPE/tmp/mplayer_cmdclient → mplayer 指令通道
TMP_SONG_FILE/tmp/carradio_song.mp3下载缓存文件
STATUS_PIPE/tmp/carradio_statusclient → UI 状态通道

5.3 三条命名管道

管道写端读端内容示例
/tmp/ui_cmdQt UIclient_appplay:5, next
/tmp/mplayer_cmdclient_appmplayerpause, seek 100 2
/tmp/carradio_statusclient_appQt UIPLAYING:5, IDLE

5.4 函数详解

static void on_signal(int sig)

进程退出时 kill mplayer 子进程，删除临时文件，_exit(0)。

void create_fifo(const char *path)

创建命名管道（mkfifo(path, 0666)），已存在则跳过。

static void write_status(int fd, const char *fmt, ...)

向 status 管道写入格式化状态，支持 printf 风格可变参数，尾部自动加 \n。

static int parse_cmd(char *raw, int n, int *target, int mplayer_fd, int verbose)

返回值指令示例含义
1play:5点歌 (*target = 5)
2next下一首
3quit退出系统
0pause转发给 mplayer
-1无效错误提示

int main() — 主程序

while (running) {
  阶段1: 预加载
    connect(server) -> send("5")
    select(sockfd, ui_fd)       // 同时监听网络+UI
    recv -> write(临时文件)      // 下载
    [可中断: 用户切歌则abort]

  阶段2: 播放
    fork() -> execlp("mplayer", "-slave", "-input", ...)
    select(ui_fd, 0.5s超时)     // 监听UI指令
    waitpid(mplayer)             // 播完自动下一首

    kill(mplayer) + waitpid()    // 回收子进程
    unlink(临时文件)              // 清理缓存
}

5.5 关键技术详解

select() 多路复用

下载时同时监听网络 socket 和 UI 管道，用户切歌可立即中断下载。

fork() + execlp()

子进程变身为 mplayer，参数 -slave（从属模式）+ -input file=/tmp/mplayer_cmd。

waitpid 进程回收

waitpid(mplayer_pid, NULL, WNOHANG) 非阻塞检查，防止僵尸进程。

7. 新增：本地播放模式

7.1 设计目标

在不启动 server_app、没有 TCP 网络的场景下，client_app 能直接播放本地 media/ 目录下的 MP3 文件。

7.2 核心改动

双模切换 (online_mode 标志)

int current_song = 1, running = 1, online_mode = 1;
  • online_mode = 1：在线模式，连接服务器下载播放
  • online_mode = 0：本地模式，直接播本地文件
  • 连接失败时自动回退
  • UI 可发 mode:online / mode:local 手动切换

本地歌曲扫描函数

函数说明
count_mp3_local()统计子目录 .mp3 数量
fill_mp3_local()填入完整路径到歌曲表
mp3cmp_local()qsort 排序回调
build_local_table()构建完整路由表（ch1/ch2/ch3）
free_local_table()释放内存
play_local()fork mplayer 播放本地文件

新增状态消息

消息说明
LOCAL_PLAYING:N正在播放本地第 N 首歌
LOCAL_MODE已切换到本地模式
ONLINE_MODE已切换到在线模式

新增 UI 指令

指令说明
mode:local切换到本地播放模式
mode:online切换到在线播放模式

7.3 使用场景

# 场景 1：不开 server_app，纯本地播放
cd ~/emb260316/UNIX/CarRadio
./client_app
# → 连接失败 → 自动切换本地模式 → 直接播歌

# 场景 2：在线中切本地
echo "mode:local" > /tmp/ui_cmd

# 场景 3：本地切回在线（需 server_app 在运行）
echo "mode:online" > /tmp/ui_cmd
